### 1.2 核心元件職責

| 元件 | 職責 |
|-----|------|
| StrategyEngine | 策略執行引擎介面，定義執行契約 |
| DefaultStrategyEngine | 預設實作，協調各評估器執行 |
| StrategyEvaluatorRegistry | 評估器註冊表，管理所有評估器 |
| ConditionEvaluator | 評估單一條件是否成立（支援 AND/OR 巢狀） |
| ConfidenceScoreCalculator | 根據公式計算信號信心度 |
| SignalGenerator | 根據條件評估結果產生信號 |
| DiagnosticsCollector | 收集執行診斷資訊 |
| FactorDataLoader | 從 M07/M08/M09 載入因子數據 |
| ConditionOperatorRegistry | 條件運算子註冊表（支援擴充） |

### 1.3 執行計劃（StrategyExecutionPlan）

```java
public class StrategyExecutionPlan {
    private String strategyId;                     // 策略 ID
    private LocalDate executionDate;               // 執行日期
    private StockUniverse stockUniverse;           // 股票範圍
    private boolean includeFactorValues = true;    // 是否包含因子值
    private boolean includeDiagnostics = true;     // 是否包含診斷
    private boolean saveResults = true;            // 是否儲存結果
    private boolean parallelExecution = true;      // 是否平行執行
    private int batchSize = 100;                   // 批次大小

    // 預設計劃：全市場執行
    public static StrategyExecutionPlan fullMarket(String strategyId) {
        return StrategyExecutionPlan.builder()
            .strategyId(strategyId)
            .executionDate(LocalDate.now())
            .stockUniverse(StockUniverse.TWSE_ALL)
            .includeFactorValues(true)
            .includeDiagnostics(true)
            .saveResults(true)
            .parallelExecution(true)
            .batchSize(100)
            .build();
    }

    // 快速計劃：單一股票測試
    public static StrategyExecutionPlan singleStock(String strategyId, String stockId) {
        return StrategyExecutionPlan.builder()
            .strategyId(strategyId)
            .executionDate(LocalDate.now())
            .stockUniverse(StockUniverse.ofStocks(stockId))
            .includeFactorValues(true)
            .includeDiagnostics(true)
            .saveResults(false)
            .parallelExecution(false)
            .build();
    }
}
```

### 2.2 條件評估演算法

```
演算法：遞迴評估條件樹

輸入：
  - condition_node: 條件節點（可能是單一條件或巢狀邏輯）
  - factor_data: 因子數據 Map<factor_id, value>

輸出：
  - is_matched: 條件是否成立
  - matched_details: 匹配詳情列表

演算法：
┌────────────────────────────────────────────────────────────┐
│ function evaluate(condition_node, factor_data):            │
│                                                            │
│   // 判斷是否為巢狀邏輯節點                                  │
│   if condition_node.logic exists:                          │
│       // 巢狀邏輯節點（AND / OR）                            │
│       results = []                                         │
│       for each child in condition_node.conditions:         │
│           child_result = evaluate(child, factor_data)      │
│           results.append(child_result)                     │
│                                                            │
│       if condition_node.logic == "AND":                    │
│           is_matched = all(results are matched)            │
│       else:  // OR                                         │
│           is_matched = any(results are matched)            │
│                                                            │
│       return (is_matched, flatten(results.details))        │
│                                                            │
│   else:                                                    │
│       // 葉節點（單一條件）                                  │
│       factor_value = factor_data[condition_node.factor_id] │
│       operator = condition_node.operator                   │
│       threshold = condition_node.value                     │
│                                                            │
│       is_matched = apply_operator(                         │
│           factor_value, operator, threshold                │
│       )                                                    │
│                                                            │
│       detail = {                                           │
│           factor_id: condition_node.factor_id,             │
│           factor_value: factor_value,                      │
│           operator: operator,                              │
│           threshold: threshold,                            │
│           matched: is_matched                              │
│       }                                                    │
│                                                            │
│       return (is_matched, [detail])                        │
└────────────────────────────────────────────────────────────┘

運算子實作：
┌────────────────────────────────────────────────────────────┐
│ function apply_operator(value, operator, threshold):       │
│                                                            │
│   switch operator:                                         │
│       case "EQUAL":                                        │
│           return value == threshold                        │
│       case "NOT_EQUAL":                                    │
│           return value != threshold                        │
│       case "GREATER_THAN":                                 │
│           return value > threshold                         │
│       case "GREATER_THAN_EQUAL":                           │
│           return value >= threshold                        │
│       case "LESS_THAN":                                    │
│           return value < threshold                         │
│       case "LESS_THAN_EQUAL":                              │
│           return value <= threshold                        │
│       case "BETWEEN":                                      │
│           return threshold.min <= value <= threshold.max   │
│       case "IN":                                           │
│           return value in threshold (array)               │
│       case "CROSS_ABOVE":                                  │
│           // 需要歷史資料                                   │
│           return prev_value < threshold and                │
│                  curr_value >= threshold                   │
│       case "CROSS_BELOW":                                  │
│           return prev_value > threshold and                │
│                  curr_value <= threshold                   │
└────────────────────────────────────────────────────────────┘
```

**評估範例**:

```json
// 策略條件定義
{
  "logic": "AND",
  "conditions": [
    {"factor_id": "M07_RSI_14", "operator": "LESS_THAN", "value": 30},
    {
      "logic": "OR",
      "conditions": [
        {"factor_id": "M09_FOREIGN_NET", "operator": "GREATER_THAN", "value": 0},
        {"factor_id": "M09_TRUST_NET", "operator": "GREATER_THAN", "value": 0}
      ]
    }
  ]
}

// 股票 2330 的因子數據
{
  "M07_RSI_14": 25.3,
  "M09_FOREIGN_NET": 5000000,
  "M09_TRUST_NET": -200000
}

// 評估過程
1. 評估 RSI < 30: 25.3 < 30 = true
2. 評估巢狀 OR:
   - 外資 > 0: 5000000 > 0 = true
   - 投信 > 0: -200000 > 0 = false
   - OR 結果: true (至少一個成立)
3. AND 結果: true AND true = true

// 輸出
{
  "is_matched": true,
  "matched_details": [
    {"factor_id": "M07_RSI_14", "value": 25.3, "matched": true},
    {"factor_id": "M09_FOREIGN_NET", "value": 5000000, "matched": true},
    {"factor_id": "M09_TRUST_NET", "value": -200000, "matched": false}
  ]
}
```

### 2.3 信心度計算演算法

```
演算法：基於公式計算信心度分數

輸入：
  - confidence_formula: 信心度公式字串
  - factor_data: 因子數據 Map
  - matched_conditions: 匹配條件詳情

輸出：
  - confidence_score: 信心度分數 (0-100)

公式語法支援：
  - 變數: 因子 ID（如 RSI, PE, foreign_net）
  - 運算子: +, -, *, /
  - 函數: MIN(), MAX(), ABS(), NORMALIZE()
  - 常數: 數字

範例公式：
┌────────────────────────────────────────────────────────────┐
│ "(30 - RSI) / 30 * 0.4 + (20 - KD_K) / 20 * 0.3 +         │
│  chip_score / 100 * 0.3"                                   │
│                                                            │
│ 解讀：                                                     │
│ - RSI 越低（超賣），分數越高（權重 40%）                      │
│ - KD 越低（超賣），分數越高（權重 30%）                       │
│ - 籌碼評分越高，分數越高（權重 30%）                          │
└────────────────────────────────────────────────────────────┘

演算法：
┌────────────────────────────────────────────────────────────┐
│ function calculate_confidence(formula, factor_data):       │
│                                                            │
│   // 1. 解析公式                                            │
│   expression = parse_expression(formula)                   │
│                                                            │
│   // 2. 替換變數                                            │
│   for each variable in expression:                         │
│       if variable in factor_data:                          │
│           replace variable with factor_data[variable]      │
│       else:                                                │
│           // 變數不存在，使用預設值或報錯                     │
│           replace variable with 0 or throw error           │
│                                                            │
│   // 3. 評估表達式                                          │
│   raw_score = evaluate_expression(expression)              │
│                                                            │
│   // 4. 正規化至 0-100 範圍                                 │
│   confidence_score = MAX(0, MIN(100, raw_score * 100))     │
│                                                            │
│   return confidence_score                                  │
└────────────────────────────────────────────────────────────┘
```

## 5. 快取策略

### 5.1 快取層級

```
┌─────────────────────────────────────────────────────────────────┐
│                        快取架構                                   │
└─────────────────────────────────────────────────────────────────┘

         請求
          │
          ▼
┌─────────────────────┐
│   L1: 本地快取       │ ← Caffeine Cache
│   - 策略定義         │     TTL: 30 分鐘
│   - 因子元數據       │     Size: 100 entries
└─────────────────────┘
          │
          │ 未命中
          ▼
┌─────────────────────┐
│   L2: Redis 快取    │ ← 分散式快取
│   - 策略定義         │     TTL: 24 小時
│   - 因子數據         │     TTL: 6 小時
│   - 執行結果         │     TTL: 1 小時
└─────────────────────┘
          │
          │ 未命中
          ▼
┌─────────────────────┐
│   L3: PostgreSQL    │ ← 持久化儲存
│   - 完整資料         │
└─────────────────────┘
```

### 5.2 快取 Key 設計

| 快取類型 | Key Pattern | TTL | 說明 |
|---------|-------------|-----|------|
| 策略定義 | `strategy:def:{strategyId}` | 24 小時 | 策略變更頻率低 |
| 策略版本 | `strategy:ver:{strategyId}:{version}` | 永久 | 版本不可變 |
| 因子元數據 | `factor:meta:all` | 24 小時 | 因子定義不常變 |
| 因子數據 | `factor:data:{stockId}:{date}` | 6 小時 | 當日計算結果 |
| 執行結果 | `strategy:exec:{executionId}` | 1 小時 | 最近執行結果 |
| 策略信號 | `strategy:signals:{strategyId}:{date}` | 1 小時 | 當日信號快取 |

### 6.1 執行錯誤處理

```
┌─────────────────────────────────────────────────────────────────┐
│                    錯誤處理流程                                   │
└─────────────────────────────────────────────────────────────────┘

     正常執行流程
          │
          ├──────────────────────────────────────┐
          │                                      │
          ▼                                      ▼
   ┌─────────────────┐                 ┌─────────────────┐
   │  單股計算失敗    │                 │   批次整體失敗   │
   └─────────────────┘                 └─────────────────┘
          │                                      │
          ▼                                      ▼
   ┌─────────────────┐                 ┌─────────────────┐
   │ Continue 模式   │                 │  執行記錄標記    │
   │ - 記錄失敗原因   │                 │  status=FAILED  │
   │ - 跳過此股票     │                 │  error_message  │
   │ - 繼續下一檔     │                 └─────────────────┘
   └─────────────────┘                          │
          │                                      ▼
          ▼                            ┌─────────────────┐
   ┌─────────────────┐                 │   觸發告警      │
   │ 統計失敗數      │                 │  - 失敗率 > 10%  │
   │ - diagnostics   │                 │  - 連續失敗 >= 3 │
   │   .skipped     │                 └─────────────────┘
   └─────────────────┘

常見錯誤類型：
┌────────────────────────────────────────────────────────────────┐
│ 錯誤類型              │ 處理方式                                 │
├───────────────────────┼─────────────────────────────────────────┤
│ 因子數據缺失          │ 跳過此股票，記錄到 diagnostics           │
│ 條件評估異常          │ 跳過此股票，記錄錯誤                      │
│ 信心度計算異常        │ 使用預設值 0，記錄警告                    │
│ 資料庫寫入失敗        │ 重試 3 次，仍失敗則標記執行失敗           │
│ 執行逾時              │ 中斷執行，標記 status=TIMEOUT            │
└────────────────────────────────────────────────────────────────┘
```
