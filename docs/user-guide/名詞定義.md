## 1. 業務術語

### 1.1 股票相關

| 術語 | 英文 | 定義 | 範例 |
|-----|------|-----|------|
| **股票代碼** | Stock ID / Symbol | 台股股票代碼，4 位數字 | 2330（台積電）、0050（元大台灣50） |
| **股票名稱** | Stock Name | 股票的中文名稱 | 台積電、鴻海、聯發科 |
| **上市** | Listed | 在台灣證券交易所（TWSE）掛牌交易的股票 | 台積電（2330）在證交所上市 |
| **上櫃** | OTC (Over-The-Counter) | 在櫃檯買賣中心（TPEx）交易的股票 | 中小型股票多在櫃買中心掛牌 |
| **產業別** | Industry / Sector | 股票所屬的產業分類 | 半導體業、金融業、傳產業 |
| **市值** | Market Cap | 股票總市值 = 股價 × 流通股數 | 台積電市值約 15 兆台幣 |
| **成交量** | Volume | 當日股票交易的總股數 | 今日成交量 10,000 張 |
| **自選股** | Watchlist | 使用者關注的股票清單 | 我的自選股：2330、2454、2317 |

---

### 1.2 技術分析相關

| 術語 | 英文 | 定義 | 範例 |
|-----|------|-----|------|
| **技術指標** | Technical Indicator | 基於股價、成交量計算的數學指標 | MA、MACD、RSI、KD |
| **均線** | Moving Average (MA) | 一段時間內股價的平均值 | MA5（5 日均線）、MA20（20 日均線） |
| **黃金交叉** | Golden Cross | 短期均線向上穿越長期均線 | MA5 向上突破 MA20 |
| **死亡交叉** | Death Cross | 短期均線向下穿越長期均線 | MA5 向下跌破 MA20 |
| **超買** | Overbought | 股價上漲過快，可能面臨回檔 | RSI > 70 表示超買 |
| **超賣** | Oversold | 股價下跌過快，可能出現反彈 | RSI < 30 表示超賣 |
| **支撐線** | Support Level | 股價不易跌破的價位 | 股價在 100 元有強力支撐 |
| **壓力線** | Resistance Level | 股價不易突破的價位 | 股價在 120 元遇到壓力 |
| **突破** | Breakout | 股價突破壓力線或重要關卡 | 股價突破 120 元壓力 |
| **技術型態** | Pattern | K線組合或圖表形成的特定型態 | 頭肩頂、雙重底、三角收斂 |

---

### 1.3 基本面分析相關

| 術語 | 英文 | 定義 | 範例 |
|-----|------|-----|------|
| **基本面指標** | Fundamental Indicator | 基於財報計算的財務指標 | EPS、ROE、本益比 |
| **EPS** | Earnings Per Share | 每股盈餘 = 淨利 ÷ 流通股數 | EPS = 8.5 元 |
| **本益比** | P/E Ratio | 股價 ÷ EPS，評估股價是否合理 | 本益比 = 15 倍 |
| **ROE** | Return on Equity | 股東權益報酬率 = 淨利 ÷ 股東權益 | ROE = 18% |
| **營收** | Revenue | 公司的營業收入 | 月營收 100 億元 |
| **毛利率** | Gross Margin | (營收 - 成本) ÷ 營收 | 毛利率 = 40% |
| **淨利率** | Net Profit Margin | 淨利 ÷ 營收 | 淨利率 = 15% |
| **負債比** | Debt Ratio | 負債 ÷ 總資產 | 負債比 = 35% |
| **股利** | Dividend | 公司分配給股東的盈餘 | 現金股利 2 元 |
| **殖利率** | Dividend Yield | 股利 ÷ 股價 | 殖利率 = 4% |

### 1.4 籌碼分析相關

| 術語 | 英文 | 定義 | 範例 |
|-----|------|-----|------|
| **籌碼** | Chip | 市場參與者的買賣籌碼 | 三大法人、散戶、主力 |
| **三大法人** | Three Major Institutional Investors | 外資、投信、自營商 | 外資買超 5,000 張 |
| **外資** | Foreign Investors | 境外機構投資人 | 外資持股比例 75% |
| **投信** | Investment Trust | 本國投資信託公司 | 投信買超 2,000 張 |
| **自營商** | Dealers | 證券商自有資金買賣部門 | 自營商賣超 500 張 |
| **融資** | Margin Trading | 向券商借錢買股票 | 融資餘額 10,000 張 |
| **融券** | Short Selling | 向券商借股票賣出 | 融券餘額 1,000 張 |
| **融資維持率** | Margin Maintenance Ratio | 融資擔保品市值 ÷ 融資金額 | 融資維持率 130% |
| **董監持股** | Directors & Supervisors Holdings | 公司董事、監察人持有的股份 | 董監持股比例 15% |
| **籌碼集中度** | Chip Concentration | 大股東持股的集中程度 | 前十大股東持股 60% |

### 1.5 信號與策略相關

| 術語 | 英文 | 定義 | 範例 |
|-----|------|-----|------|
| **信號** | Signal | 系統根據指標/型態/策略產生的買賣建議 | 黃金交叉產生買進信號 |
| **進場信號** | Entry Signal / Buy Signal | 建議買入的信號 | RSI 低於 30，產生進場信號 |
| **出場信號** | Exit Signal / Sell Signal | 建議賣出的信號 | 股價跌破支撐線，產生出場信號 |
| **選股** | Stock Screening | 根據條件篩選出符合的股票清單 | 高 ROE + 低本益比選股 |
| **策略** | Strategy | 一組投資規則與邏輯的組合 | 動能策略、價值投資策略 |
| **回測** | Backtest | 使用歷史資料驗證策略的有效性 | 回測過去 5 年的績效 |
| **績效** | Performance | 投資策略的獲利表現 | 年化報酬率 15% |
| **停損** | Stop Loss | 當虧損達到特定比例時自動賣出 | 設定停損 -10% |
| **停利** | Take Profit | 當獲利達到特定比例時自動賣出 | 設定停利 +20% |
| **警報** | Alert | 滿足特定條件時觸發的通知 | 股價突破 120 元時發送警報 |

### 1.6 投資組合相關

| 術語 | 英文 | 定義 | 範例 |
|-----|------|-----|------|
| **投資組合** | Portfolio | 投資者持有的所有股票的集合 | 我的投資組合包含 10 檔股票 |
| **部位** | Position | 單一股票的持有數量與成本 | 台積電部位：100 張，成本 500 元 |
| **持股** | Holdings | 目前持有的股票 | 持股：2330、2454、2317 |
| **損益** | P&L (Profit & Loss) | 投資的獲利或虧損 | 今日損益 +5,000 元 |
| **報酬率** | Return Rate | 獲利 ÷ 投入成本 | 報酬率 = 10% |
| **風險** | Risk | 投資可能虧損的程度 | 高風險高報酬 |
| **資產配置** | Asset Allocation | 資金分配到不同資產的比例 | 股票 60%、債券 40% |

## 2. 技術術語

### 2.1 Spring Boot 相關

| 術語 | 定義 | 用途 |
|-----|-----|------|
| **Entity** | JPA 實體類別，對應資料庫表 | `@Entity` 標註的 Java 類別 |
| **DTO** | Data Transfer Object，資料傳輸物件 | 用於 API 請求/回應，與 Entity 分離 |
| **Repository** | JPA 資料存取層接口 | 負責基礎 CRUD 操作 |
| **Mapper** | MyBatis Mapper 接口 | 用於複雜查詢和動態 SQL |
| **Service** | 業務邏輯層 | 處理核心業務規則，協調 Repository 和 Mapper |
| **Controller** | 控制器層 | 處理 HTTP 請求與回應 |
| **Scheduler** | 定時任務 | 用於週期性執行資料更新、信號檢測等 |
| **Configuration** | 配置類別 | 系統配置與 Bean 定義 |

---

### 2.2 資料庫相關

| 術語 | 定義 | 範例 |
|-----|-----|------|
| **Schema** | 資料庫結構定義 | 包含表、欄位、索引、關聯 |
| **Table** | 資料表 | `stocks`, `stock_prices`, `signals` |
| **PK (Primary Key)** | 主鍵 | `stock_id` 是 stocks 表的主鍵 |
| **FK (Foreign Key)** | 外鍵 | `stock_id` 是 stock_prices 表的外鍵 |
| **UK (Unique Key)** | 唯一鍵 | `signal_uuid` 必須唯一 |
| **IDX (Index)** | 索引 | 加速查詢的資料結構 |
| **JSONB** | PostgreSQL 的二進制 JSON 型態 | 支援高效索引與查詢 |
| **Transaction** | 交易 | 確保多個操作的原子性 |
| **ACID** | 原子性、一致性、隔離性、持久性 | 資料庫交易的四大特性 |

---

### 2.3 系統架構相關

| 術語 | 定義 | 說明 |
|-----|-----|------|
| **Job** | 可追蹤的批次作業單位 | 有狀態、可重跑、可追溯 |
| **Pipeline** | 資料處理流水線 | 包含多個步驟的資料處理流程 |
| **Cache** | 快取 | 使用 Redis 儲存熱資料以提升效能 |
| **冪等性** | Idempotency | 同一操作執行多次結果一致 |
| **事件驅動** | Event-Driven | 透過事件解耦模組間的依賴 |
| **負載均衡** | Load Balancing | 分散流量到多個伺服器 |
| **水平擴展** | Horizontal Scaling | 增加伺服器數量來擴展系統 |
| **垂直擴展** | Vertical Scaling | 增加單一伺服器的硬體資源 |

## 📝 維護說明

### 新增術語流程

1. 確認術語類別（業務/技術/縮寫）
2. 填寫術語定義、英文對照、範例
3. 更新相關文檔的連結
4. 通知團隊成員

### 術語統一原則

- ✅ 使用業界標準術語
- ✅ 中英文對照清楚
- ✅ 提供實際範例
- ✅ 定義簡潔明確
- ❌ 避免自創術語
- ❌ 避免過於冗長的定義
